# Section: Results

Results should report what was observed before interpreting why it occurs.

## Advisor-Derived Results Rules

- Use short, object-focused subsection titles when the object is unambiguous.
- Open results subsections with the studied object, adjusted parameters, and the physical dimension probed by each parameter.
- Avoid long mechanism explanations before the observed trend is established.
- Do not add cross-subsection bridge sentences unless they carry essential logic.
- For figures with curves and field/distribution plots, discuss curves first, then damage/stress/plastic-dissipation fields, then give a compact mechanism summary.
- Inspect corresponding figure assets whenever available before interpreting figure trends, panel order, field-variable names, units, or spatial localization.
- Keep paragraph-level summary sentences when they close a result unit, but compress them to the controlling relation.
- Delete low-information figure narration, especially color keys or panel descriptions obvious from the figure.
- Use case-based grouping to reduce repetition.
- Preserve supported mechanism terms such as alignment, axial bridging, deformation redistribution, geometric continuity, damage field, and bond failure.
- If field plots omit or mask CNT-interior values for visualization, state this once as a visualization choice.
- Use `bending offset parameter q` for the parameter definition; use `curvature` only as morphology description unless the author changes the parameter system.
- Use `damage field`, `damage evolution`, `damage bands`, `bond failure`, and `fracture process` according to scale and mechanism.
- Use `Figure` at sentence start and LaTeX spacing such as `Figure~\ref{...}(a)` when editing LaTeX.
